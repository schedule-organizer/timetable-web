# Epic 1 - Platform Foundation & Secure Access

**Goal:** Establish the core application shell, authentication, authorization, and shared UX infrastructure required for every subsequent feature.

**Primary personas:** Timetabler, Principal, Teacher, Student, Parent

**Business value:**
- makes the product trial-ready
- supports multi-role access across the school community
- reduces early trust and security concerns

**Key capabilities:**
- registration, login, logout, session refresh
- role-based route protection
- multi-role user accounts with active role switching
- profile management and teacher invite completion
- base design system, i18n, state management, and app shell

**Frontend alignment:** `FOUND-02`, `FOUND-10` to `FOUND-15`, `AUTH-11` to `AUTH-16`

**Success signal:** a new institution can register, invite staff, and access role-appropriate areas without manual support.
