# Epic 3 - Core Resource Management

**Goal:** Capture and maintain the core entities the scheduling engine depends on.

**Primary personas:** Timetabler, Admin

**Business value:**
- creates the operational data backbone of the product
- replaces spreadsheet-based setup with structured, reusable records
- enables consistent scheduling and reporting

**Key capabilities:**
- rooms and room constraints
- subjects and difficulty metadata
- classes and required subject hours
- teacher records, qualifications, and availability
- teaching groups and option blocks

**Frontend alignment:** `RES-12` to `RES-17`

**Success signal:** an institution can model the full set of teachers, classes, subjects, rooms, and grouping rules needed for timetable generation.
